## ReadMe
原則として一度のプロデュースで二度同じコミュは表示されません。  
ただしコミュが枯渇した場合のみ、ランダムから選ばれます。  
また、一部のコミュを除いて基本的に数が決まっていません。目安としてアケ・箱の情報を載せていますが、各作者の思うように書いていただいて構いません。

### 表示頻度
**ランダム>ある日>季節** になっています。  
ランクアップコミュはランクアップ後の週のみ確定表示です。そのときにはおすすめ表示がされる予定です。  
休日コミュは休みコマンド選択時、1/2の確率で発生します。(複数用意されていた場合はその中からランダム)

### ラストコンサートコミュ
~~ラストコンサートの実装は相当先になります。~~  
~~そのため、アイドルランク依存のエンディングを用意していただくことになっています。~~  
~~エンディングの実行条件は、~~
  1. ~~目標週にランクアップできなかった~~
  2. ~~61週のプロデュースを終えた~~

~~のどちらかです。~~  
~~ラストコンサートコミュに関しては、プロデュース中のデータでもセーブデータを修正することで問題なくプレイ出来るように予定していますので、プロデューサー各位にはご迷惑をおかけしますがよろしくお願いします。~~  
~~セーブデータの修正は一般的なテキストエディタで行うことができますので、近くなりましたらアナウンスさせていただきます。~~  

追記：**ラストコンサート** の実装、します。  
いろいろ考えて、エンディングまでちゃんと動かない状態じゃちょっとダメななと思って。それにそこまで実装が難しいかというと、おそらくオーディションシステムさえ作れたら似た実装をすれば動くと思うので、頑張ってみようと思います。  
細かなテキスト仕様はまた公開しますが、大雑把に言うと
1. 活動中止が決定(ランクアップできないなど)
2. 会場を選ぶ
3. 衣装、アクセ、楽曲を選ぶ(オーディションと同様)
4. コンサート前、最後の会話(特定衣装やアクセで特別な会話イベントにしたいものの、仕様の制約上厳しそう？考えてみる)
5. 特定の計算式による判定でライブの成功、失敗を判断。
6. その結果に合わせて最後のコミュ。

5番の実装がどうなるか全くわからないですが、レッスン・オーディションと似たような感じになるはず。  
テキストの話は下記、コミュ解説にて行います。

### Zエンドについて
プロデュース開始時に、Zエンドを見るかどうかの確認が有ります。  
ZエンドはFランクと同じ位――要するに、一度もアイドルランクアップできなかったということです。  
原則コミュに関して制約はありませんが、Zエンドのみは**プロデューサーの心に傷を残すようなコミュでなければならない** と考えていてください。  
Zエンドは必須ではありません。そのアイドルがどうであるか、作者自身が書けるかなどを考慮してください。

## Constraints
選択肢は**最大五個** です。それ以上用意しても以降の選択肢は表示されません。  
異なる選択肢から同じ分岐先に行くことも可能です。  
選択肢でなくとも、あるインデックスに飛ぶことも可能です。

    例：【】は選択肢、[]は飛ぶ先  
    index1:「～～～～」「～～～～」「～～～」【[index2]or[index3]】  
    index2:「～～～」「～～～」「～～～」「～～～」[index4]  
    index3:「～～～」「～～～」[index4]  
    index4:「～～～」「～～～」「～～～」[end]

## コミュ解説
※解説内に書いてあるシチュエーションは自由に使ってくださって構いません。

### first
初対面コミュ  
シチュエーションについての指定はありません。  
事務所にいたアイドルのもとにプロデューサーが来た設定でも、プロデューサーがアイドルをスカウトした設定でも、プロデューサーのいる事務所にアイドルが来た設定でも問題ありません。  
あえていうと毎回プロデュースするたびに見るシーンなので、特別な思いを込めてあげてほしいなと思います。

### meeting
ミーティングコミュ  
アイドルと仕事の場でしっかり話す初めての機会です。  
例えばそのアイドルがどんなものを目指しているのか、レッスン方針はどうなのか、なんかを話せたら第一歩がようやく踏み出せたような気がするのではないでしょうか。

### composer
作曲家挨拶コミュ  
初めて対外するシーン。  
アイドルは緊張するでしょうか？ワクワクしているでしょうか。そのアイドルの芯の部分が見えてくるのかもしれません。

---

### random
ランダムコミュ。所謂通常営業のこと。  
それぞれのランクごとに用意していただきますが、複数ランクで同じコミュにしても大丈夫です。  
本家ではCランクまでは**ミーティング+8~9種類** が用意されていて、Bランク以降はまとめてミーティング+8種類用意されている。  
基本的に上位になるほど規模の大きな営業を行う事が多いので、それに合わせてみるのもいいと思います。  
もちろんパクリはダメですよ。

### rankup
ランクアップコミュ  
気合い入れて書いてもらいたい部分です。E~Sまで、ステップを踏んでいく姿が見れたらプロデューサー冥利に尽きるというか。  
弱い一面を見せてもいいし、段々と実力を実感してきて野心をむき出しにする姿を見せてもいいと思います。

### oneday
ある日のコミュ。各ランクごとにお願いします。  
本家ではF,Eが一種類、D,Cが二種類、B以降まとめて一種類で、合計七種類が用意されているので、それを目安にしていただければよいと思います。    
下記のコミュとのバランスなどを考慮して、**事務所で起こった仕事以外のイベント** もここで書くとちょうどいいかもしれません。  
もちろん、昼休み公園で悩み事をしているようなシーンだったり、一緒に帰りませんか？と言われて夕食を一緒に食べに行くなんてシーンもGoodです。

### season
季節のコミュ  
基本的に12週ごとにローテーションされる。全四種。ただし最初の12週には起こらず、60週以降も起こらない。  
プロデュース開始の時期などは指定がないので、本家のように人口スキー場やらとしてプレイヤーに委ねても良いし、ある時期からと決めてもいいと思います。  
誕生日などの特別なイベントを考えている場合、それに合わせてプロデュースの開始時期を決めてもいい気はします。  
若干ご都合主義的なので、好き嫌いが分かれるかもしれませんが。

### holiday
休日コミュ  
発生自体が休みコマンド選択時に確率なのでレアコミュです。(狙おうと思うとすごく簡単になりますが)  
休日ということもありますし、上記よりももっと私生活的な面を取り上げてもらえるとアイドルの理解がよりいっそう深まると思います。  
もちろんアイドルによっては休日も気を張っているかもしれないので、そのあたりはそれぞれのアイドルと相談して書いていただけると良いコミュになると思います。

### ending
エンディングコミュ  
このときはやってきます。最後に待っているのはラストコンサート。  
アイドルとの信頼をすべて出し切って、最高のラストを飾れるのでしょうか。

まず基礎として、コンサート前とコンサート後の二度コミュイベントがあります。  
以下のテーブルに乗っ取り、どちらかの会場でコンサートをやることになります。

|アイドルランク|会場１|会場２|
|--|---|--|
|E以下|ライブハウス|市民ホール |
|D|市民ホール|アリーナ|  
|C|アリーナ|武道館|
|B以上|武道館|ドーム|

それぞれの会場、ランクでコミュを変えるかは自由です。  
さらに、Sランクのドームではトゥルーエンドが用意可能です。  
よって最大では8x1=8種と8x2=16種+1種必要になります。最低では5x1=5種と5x2=10種+1種となります。  
会場が同じであるから基本的に同じ結末をたどると考えるかは、アイドルをじっっっっっくりと考えて悩んでください。

## Memo(For Framework Developer)
### random
コミュの数に制約はないが、増やしすぎると特定のランダムを当てるのが大変になるのでコンプしにくくなる。  
ランクごとに分けるため、各ランクをキーとして連ねていく。  
例:

    {
        "random": {
            "F": [{
                "main": ["P", "FランクランダムA"]
            }, {
                "main": ["P", "FランクランダムB"]
            }],
            "E": [{
                "main": ["P", "EランクランダムA"]
            }, {
                "main": ["P", "EランクランダムB"]
            }],
            //以下略//
        }
    }
